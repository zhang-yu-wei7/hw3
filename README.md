##4. 人工代码审查（任务 4 PHF 计算核心逻辑）
在任务 4 中，PHF（高峰小时系数）的计算需要精确的时间重采样。AI 最初使用了过时的时间频率字符串，我进行了人工修正。以下是最终确认的核心代码逻辑：
# 1. 找出客流量最大的“高峰小时”
# 使用 groupby 按“小时”分组，size() 统计每组的记录数（即该小时的刷卡量）
hourly_counts = df.groupby('小时').size()

# idxmax() 获取流量最大的那个小时的索引值（即具体是几点）
peak_hour = hourly_counts.idxmax()

# max() 获取该高峰小时的流量总数
peak_hour_volume = hourly_counts.max()

# 2. 筛选高峰小时内的数据并进行时间重采样
# 将交易时间设为索引，以便使用 resample
# copy() 是为了避免 SettingWithCopyWarning 警告
df_peak = df[df['小时'] == peak_hour].copy()
df_peak.set_index('交易时间', inplace=True)

# 5分钟粒度统计：使用 '5min' 进行重采样
# size() 统计每个 5 分钟窗口内的记录数
counts_5min = df_peak.resample('5min').size()

# 计算 PHF5 (高峰小时流量 / (12 * 最大 5 分钟流量))
# 1 小时有 12 个 5 分钟，所以系数是 12
# max() 获取这 12 个窗口中流量最大的那个值
phf5 = peak_hour_volume / (12 * counts_5min.max())

# 打印结果，保留 4 位小数
print(f"高峰小时: {peak_hour}:00")
print(f"PHF5: {phf5:.4f}")
